# design-examples
### Application and general visual design

Nothing in this portfolio was done using AI except the functionality behind Ask the Design System with AI. (clearly)

## [Billing Rules](https://github.com/ShiraJoseph/design-examples/blob/main/Billing%20Rules.png)
![Billing Rules](https://github.com/ShiraJoseph/design-examples/raw/main/Billing%20Rules.png)

This is one view from a full set of ~25 mockups that I designed in Figma for a cloud account management platform.  I was an individual developer contributor at the time, but the team lacked direction and the designers were all busy.  After seeing how I made one mockup a previous time, the department engineering manager asked me to help the product owners and the rest of the teams figure out what was needed for a pressing deadline.  So I gathered product owners and backend developers, got everyone to agree on requirements, and produced the complete set of mockups without help (and before AI) - in 6 days. Although the company design system already contained colors, borders, etc., the direction was mine, the drag-and-drop feature was completely new, and the rules cards were my idea.  

I then led the other front-end developers on my team to implement the entire feature (5-6 screens with sub pages) in two months, although their lack of experience meant that I wrote 85% of the code myself while still spending half of my time answering their questions on how to write the remainder.  Those were some 70hr work weeks - I don't recommend it.  But I was determined to help management, to get the product out the door, and deliver the best thing we could for our users within the given timeframe.  

## [Ask the Design System With AI](https://github.com/ShiraJoseph/design-examples/blob/main/Ask%20the%20Design%20System%20with%20AI.png)
![Ask the Design System With AI](https://github.com/ShiraJoseph/design-examples/raw/main/Ask%20the%20Design%20System%20with%20AI.png)

This is part of a POC [design-system library](https://github.com/ShiraJoseph/design-system) I made.  It is a shared component that can be paired with any AI API to answer questions about how to use the library.  By default it comes with a static catalog provider.

I think information should be easy to obtain, and I'm excited to see how this will change as AI improves.  Ideally you should be able to tell an AI what components you need and have it pull them from the library and plug them into your code; this is a helper for you to discover what is available and how to use it.

## [A Little Book About Fractions](https://github.com/ShiraJoseph/design-examples/blob/main/A%20little%20book%20about%20fractions.pdf)
PDF: [A Little Book About Fractions](https://github.com/ShiraJoseph/design-examples/blob/main/A%20little%20book%20about%20fractions.pdf)

I created this book back when I was tutoring math. I love taking concepts that people may think are confusing and consolidating them into a straightforward, step-by-step algorithm, with pretty colors and as little visual clutter as possible.  Anyone who knows how to add, subtract, multiply and divide should be able to use this book as a reference for all the Rules of Fractions.  It's a "cheat sheet" for adults who want to brush up on their own and for children when paired with practice problems.
